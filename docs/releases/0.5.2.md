# HeadMatch 0.5.2

## Summary

Bug fix and UX release focused on the target editor and headphone search. The target editor now supports live-updating sliders, canvas drag-to-move, and correct behaviour when adding/removing control points. Headphone search handles model names without spaces (e.g. "HD650" finds "HD 650").

## Fixes

- **Headphone search ignores spaces in model numbers**: "HD650" now finds "Sennheiser HD 650" via space-stripped comparison fallback. Sorting also accounts for compact prefix matches.
- **Fetch curve URL encoding**: AutoEQ URLs with spaces in model names (e.g. "Sennheiser HD 650") are now percent-encoded. Previously `urlopen` rejected them with "can't contain control characters".
- **Fetch curve fields update on re-selection**: Selecting a different headphone from search results now always updates both the URL and "Save to" fields. Previously the output path was only set on first selection.
- **APO refine crash fix**: `_run_apo_refine` referenced `self.config_path` (nonexistent); corrected to `self.state.config_path`.
- **Target editor no longer resets values when adding a point**: `ttk.Scale.set()` fires its callback during widget construction; a setup guard now suppresses sync until all widgets are built.
- **Canvas drag works smoothly**: Drag events bound to the canvas widget (not individual items), with lightweight mid-drag redraw that only updates the curve and control points. Full redraw + widget sync on release.

## Improvements

- **Target editor live updates**: Sliders update the curve preview in real-time. Frequency entries commit on Return/FocusOut. Removed the "Apply changes" button — everything is live.
- **Canvas drag-to-move**: Control points can be dragged directly on the frequency response graph with log-frequency X axis and linear dB Y axis. Dragged point highlighted in yellow.
- **Curve preview above controls**: Preview canvas moved above the control points table for immediate visual feedback.
- **Scrollable control points table**: When >8 control points exist, the table wraps in a scrollable frame.

## Architecture

- **GUI view extraction**: `render_import_apo()`, `render_fetch_curve()`, and `render_history_page()` moved from `gui.py` into `gui_views.py`. gui.py: 958 → 865 lines.
- **`_PlotGeometry` class**: Bidirectional coordinate mapping (freq↔x, dB↔y) shared by the curve renderer and drag handler. Clamped to valid ranges.

## Tests

436 → 447 deterministic tests (+11). New coverage: space-insensitive search, coordinate round-trips, clamping, canvas rendering with 2/7/15+ control points.
