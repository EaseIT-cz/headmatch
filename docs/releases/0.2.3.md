# HeadMatch 0.2.3

## Summary

HeadMatch 0.2.3 is a correctness and safety patch.

It tightens capture validation so broken recording chains are caught early instead of producing misleading EQ presets.

## Key changes

### 1. Mono captures are now rejected

Previously, a mono recording was silently duplicated to stereo and treated as valid.
This hid broken capture setups and produced identical L/R EQ that looked plausible but was meaningless.

0.2.3 rejects mono captures with a clear error message explaining what to do.

### 2. Duplicated-channel captures are now rejected

If both stereo channels are identical (indicating a broken split, loopback, or driver issue), the pipeline now detects this and rejects the capture with an actionable message instead of proceeding to fit.

This check applies to all input files regardless of channel count — 2-channel, 4-channel, or higher.

### 3. Test and documentation cleanup

- Capture validation tests use idiomatic `pytest.raises`.
- Architecture and backlog docs updated to reflect shipped GraphicEQ support and current project state.

## User impact

If you previously got a result where left and right measurements were suspiciously identical, 0.2.3 will now tell you why and ask you to fix the recording setup before proceeding.

## Scope

Patch release. No new workflow modes or features.

## Upgrade

```bash
python -m pip install --upgrade headmatch
```

Then verify:

```bash

```
