# HeadMatch 0.6.0

## Summary

Cross-platform release. HeadMatch now runs on macOS (and experimentally on Windows) via a pluggable audio backend architecture. Linux/PipeWire support is unchanged.

## macOS support

- **PortAudio audio backend**: device discovery, simultaneous play/record, and doctor checks via `sounddevice` library
- **Platform-aware paths**: config in `~/Library/Application Support/headmatch/`, cache in `~/Library/Caches/headmatch/`
- **Install**: `pip install headmatch[portaudio]`
- **Device targets**: numeric IDs or name substrings (`--output-target 2` or `--output-target "MacBook"`)

## Architecture

- **AudioBackend protocol**: abstract interface for platform-specific audio I/O
- **get_audio_backend() factory**: auto-selects PipeWire on Linux, PortAudio on macOS/Windows
- **backend_pipewire.py**: all PipeWire logic extracted from measure.py (unchanged behaviour)
- **backend_portaudio.py**: new PortAudio implementation using `sounddevice`
- **measure.py**: thin orchestration layer with backward-compatible API wrappers

## GUI improvements

- Device dropdowns show `ID — Label` (e.g. `2 — MacBook Pro Speakers`) instead of bare IDs
- Default output directory: `~/Documents/HeadMatch/session_01` (cross-platform)
- Fetched curves save to `~/Documents/HeadMatch/` by default
- Config auto-saved after every successful measurement/fit
- Desktop shortcut button hidden on non-Linux platforms

## Config migration

- New field names: `output_target` / `input_target` replace `pipewire_output_target` / `pipewire_input_target`
- Both accepted on read; new names written on save
- Old configs migrate automatically on next GUI run
- Legacy `out/session_01` default recognized and replaced

## Platform-neutral text

- GUI and CLI no longer reference PipeWire in user-facing strings
- Help text uses generic "playback device" / "capture device" terminology
- Doctor checks adapt labels to the active backend

## Fixes (carried from 0.5.2)

- Space-insensitive headphone search
- Fetch curve URL percent-encoding for spaces
- Target editor: live-updating sliders, canvas drag-to-move, no value reset on add
- APO refine config_path crash fix

## Tests

436 → 507 deterministic tests (+71). New coverage:
- 25 PipeWire backend regression tests
- 21 PortAudio backend tests (mocked)
- 8 platform path tests
- 6 config aliasing tests
- 4 coordinate mapping tests
- 3 canvas rendering tests
- 4 headphone search tests

## Upgrade

```bash
# Linux (existing users)
pip install --upgrade headmatch

# macOS (new)
pip install headmatch[portaudio]

headmatch --version
# headmatch 0.6.0
```
