# HeadMatch 0.2.0

## Highlights

HeadMatch 0.2.0 is the first solid GUI-first release of the Linux/PipeWire headphone measurement and EQ workflow.

Install from PyPI with:

```bash
python -m pip install headmatch
```

That gives you the `headmatch`, `headmatch-gui`, and `headmatch-tui` entry points directly from the published package.

This release adds:
- a cleaner GUI-first workflow with compact navigation and scrollable results
- PipeWire target dropdowns in the GUI with config/default-device preselection
- better confidence/trust presentation in both GUI and CLI
- low-confidence troubleshooting guidance in the output flow
- `headmatch doctor` for setup diagnostics
- in-app GUI setup checks
- clearer PipeWire device guidance
- example target curves and example clone-target workflows
- improved release automation via GitHub Actions, GitHub Release assets, and PyPI publishing workflow support

## Key user-facing improvements

### GUI improvements
- GUI starts directly in the guided measurement flow
- simplified sidebar labels: Measure / Setup Check / Prepare Offline / Results
- scrollable results/history content
- better target/device selection UX for PipeWire playback and capture
- more conservative theme adoption and copy cleanup

### Confidence and troubleshooting
- confidence/trust summary is surfaced in GUI and CLI
- warnings and interpretation are easier to see
- low-confidence runs now include practical troubleshooting guidance

### Setup and ergonomics
- `headmatch doctor` checks config path, PipeWire tools, discovery health, and saved target validity
- GUI includes a Setup Check surface for the same readiness path
- README includes more direct setup and launch guidance

### Example content
- example tonal target CSVs for quick experimentation
- example clone-target pairings, including:
  - FiiO JT7 → Ananda Nano
  - HD650 → HD800S
- clearer docs for the preferred personal clone workflow using your own measurements

## Important backend fixes

This release includes two important backend correctness fixes:

1. **Clone-target semantics fixed**
   - relative clone targets are now treated as relative transforms, not mistaken for absolute target curves during fitting
   - effective run targets are now resolved correctly for fitting, plotting, export, and summaries

2. **PEQ filter-budget handling fixed**
   - PEQ fitting now respects the requested `max_filters` budget even when shelf candidates are present

## Scope and caveats

HeadMatch 0.2.0 is intentionally conservative.

It is best described as:
- Linux-first
- PipeWire-focused
- GUI-first, with CLI still fully supported
- TUI maintained as a backup workflow, not a primary product surface

Clone-target examples shipped in the repo should be treated as:
- useful demonstrations
- tonal starting points
- not literal guarantees that one headphone will become another

The preferred serious clone workflow is:
- measure headphone A on your own rig
- measure headphone B on your own rig
- generate a clone target from matching-side measurement CSVs
- fit headphone A toward that generated target
