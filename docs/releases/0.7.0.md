# HeadMatch 0.7.0

## Summary

HeadMatch 0.7.0 is a GUI-focused usability and structure release.

This release introduces a beginner-oriented Basic Mode, surfaces EQ clipping guidance in both GUI and CLI flows, improves clone-target calibration documentation, and starts a deeper GUI refactor so future changes can land in smaller, safer pieces.

It also strengthens release validation with explicit coverage reporting and a CI coverage floor.

## Highlights

### Basic Mode

- Added a guided **Basic Mode** for the GUI.
- Basic Mode uses a 3-step workflow:
  1. target selection
  2. measurement
  3. review & export
- Safe defaults:
  - 48 kHz
  - 3 passes averaged
  - default playback/capture devices
  - maximum 10 PEQ filters
- Added a Basic/Advanced mode selector in the GUI sidebar.
- Basic navigation now hides advanced workflows and shows only beginner-relevant paths.

### Basic Mode target selection improvements

- Target-source controls are now conditional:
  - `flat` shows no extra controls
  - `csv` shows only the CSV picker
  - `database` shows only the database search controls
- Database search now produces a real selected target CSV instead of only a status message.
- When multiple database measurements exist for the same headphone, the user can now choose which one to use.

### Basic Mode clone-target workflow

- Added a dedicated **Clone Target** workflow inside Basic Mode.
- Lets the user build a clone-target CSV from two measurement CSV artifacts.
- Intended for headphone cloning and rig-specific mic-coloration nulling workflows.

### GUI refactoring

- Split GUI view code into per-view modules under `headmatch/gui/views/`.
- Added shared GUI helpers and services for picker/task orchestration.
- Kept compatibility layers in place while moving toward a cleaner structure.
- The GUI is still transitional, but the codebase is in a better state for follow-up decomposition.

### EQ clipping visibility

- GUI completion screens now show clipping risk, recommended preamp reduction, and headroom-loss guidance.
- CLI `fit` output now includes clipping guidance.
- Added:
  - `--show-clipping`
  - `--json` output support with `eq_clipping_assessment`

### Documentation

- Added clone-target mic calibration guidance.
- Updated backlog/architecture/docs for the Basic Mode and GUI refactor direction.
