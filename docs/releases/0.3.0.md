# HeadMatch 0.3.0

## Summary

HeadMatch 0.3.0 is a feature and correctness release addressing bugs found by specialist code review, adding DSP verification tests, improving the GUI and CLI experience, and introducing a multi-pass averaging iteration mode.

## Bug fixes

### Confidence scoring accuracy
- **`_metrics` band restriction**: Error metrics are now calculated within the 80–12000 Hz audible band instead of the full spectrum. Previously, sub-bass and ultrasonic noise inflated confidence penalties.
- **`alignment_peak_ratio` for negative offsets**: Recordings that arrive early relative to the reference sweep no longer get their confidence incorrectly penalised.

### CamillaDSP shelf filter export
- Shelf filters now export the correct Q value for CamillaDSP. Previously, the RBJ shelf slope parameter S was written directly as Q, producing incorrect filter behaviour in CamillaDSP.

### Dead code removal
- Removed `validate_stereo_audio` (io_utils.py) and `inverse_sweep` (signals.py), both orphaned and never called.

## New features

### Multi-pass averaging iteration mode
`headmatch start --iterations 3 --iteration-mode average` now measures N times, averages the frequency responses, and fits once against the averaged result. This reduces measurement noise from head position variation, ambient noise, and mic placement inconsistency. The default (`--iteration-mode independent`) preserves backward-compatible behaviour.

### CLI confidence verdict
The CLI now prints a single-line verdict as the first line of confidence output:
- `✓ This run looks trustworthy.` (green on TTY)
- `⚠ Moderate confidence — review the details below.` (yellow)
- `✗ Low confidence — check the details below.` (red)

### GUI improvements
- **Confidence badges**: History view entries now show ✓/⚠/✗ prefixes for high/medium/low confidence, making results scannable at a glance.
- **Graph display**: A "Open fit graph" button in the results view launches the SVG graph in the system viewer. Shows "Graph not available" gracefully when no graph exists.
- **Setup diagnostics**: The Setup view now uses a scrollable text widget for the doctor report, handling long reports without truncation.

## Test coverage

### Biquad coefficient verification (TASK-064 + TASK-065)
- 241 parametric reference tests verify `biquad_response_db` against RBJ cookbook formulas across a dense grid of Fc × Q × Gain × Fs × filter type.
- Extreme parameter combinations (Fc=10 Hz at 96 kHz, Q=50, ±24 dB gain, near-Nyquist) all produce finite, bounded results within 3 dB of requested gain. No stability issues found.

### Module coverage gaps closed
- `plots.py`: 3 smoke tests verify SVG generation for typical inputs, empty bands, and shelf filters.
- `signals.py`: 7 unit tests cover sweep generation, amplitude bounds, pre-silence, smoothing behaviour, and grid properties.
