# HeadMatch 0.4.1

## Summary

Bug fixes from the 0.4.0 audit, security hardening, and full GUI feature parity with the CLI.

## Bug fixes

- **iteration_mode CLI passthrough**: `--iteration-mode average` now actually works for both `start` and `iterate` commands. Previously the flag was accepted but silently ignored.
- **import-apo**: Now re-exports imported bands to APO and CamillaDSP formats instead of ignoring them.
- **Windows graph opener**: Replaced `shell=True` subprocess call with `os.startfile()` to prevent shell injection.
- **fetch-curve security**: Restricted to HTTPS-only URLs with a 5 MB response cap.
- **sdist compatibility**: Added `tests/__init__.py` so the test suite works from source tarballs.
- **Merged duplicate fit/fit-offline** CLI branches.
- **Config JSON errors** now produce user-friendly messages instead of raw `JSONDecodeError`.
- **Positive-int validation** on `--iterations` and `--max-filters` (rejects negative/zero).
- **search-headphone** now honestly states it is a placeholder.

## New GUI features

- **Target Editor view**: Build custom target curves with control points, save as CSV.
- **Import APO view**: Load and re-export Equalizer APO presets from the GUI.
- **Fetch Curve view**: Download published FR curves by URL directly in the GUI.
- **Iteration mode selector**: Choose independent or average mode in the measurement wizard.

## Test suite
392 tests, all passing.
