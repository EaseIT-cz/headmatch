# HeadMatch 0.4.0

## Summary

HeadMatch 0.4.0 is a feature release focused on performance, interoperability, and opening new use cases.

## Performance

- **Vectorised fractional-octave smoothing**: Replaced O(N²) Python loop with NumPy outer-product matrix multiply. ~50× faster at 480 points.
- **Direct biquad evaluation**: Replaced `scipy.signal.freqz` calls with inline transfer function evaluation on the unit circle. 3-5× faster per filter, compounds across the greedy fitter and joint refinement.
- **Combined effect**: Full test suite dropped from 44s to 15s (then to 24s with new tests).

## New features

### APO AutoEQ import (`headmatch import-apo`)
Load an existing Equalizer APO `.txt` parametric preset and re-optimise against your own measurement. Supports per-channel (L/R) and mono presets. Recognises PK, LS, HS, PEAKING, LSC, HSC filter type variants.

```bash
headmatch import-apo --preset community_eq.txt --recording my_recording.wav --out-dir out/reimported
```

### Headphone database integration
Two new CLI commands for working with community measurement databases:

```bash
headmatch search-headphone "HD650"
headmatch fetch-curve --url https://raw.githubusercontent.com/.../HD650.csv --out targets/hd650.csv
```

`fetch-curve` downloads a published FR curve and converts it to HeadMatch's standard CSV format. Use the result as `--target-csv` or as input to `clone-target`.

### GUI target curve editor
A new target editor view with control points and PCHIP interpolation. Users can build custom target curves visually instead of editing CSV files. The editor saves standard HeadMatch target CSVs.

The editor model (`target_editor.py`) supports:
- Add, remove, and move control points
- PCHIP (monotone cubic) interpolation on the 48 PPO grid
- Save/load from CSV
- Minimum 2 control points enforced
