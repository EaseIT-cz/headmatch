# HeadMatch 0.2.1

## Summary

HeadMatch 0.2.1 is a backend-correctness patch release focused on EQ and clone-target reliability.

This release is intended to fix cases where HeadMatch could produce clearly insufficient or mathematically misleading results even when the workflow itself looked valid.

## Key fixes

### 1. Clone-target semantics are now handled correctly

In 0.2.0, clone-target CSVs could be treated like ordinary absolute targets during fitting.
That was wrong.

0.2.1 fixes this by making clone-target semantics explicit:
- ordinary target CSVs remain **absolute targets**
- clone-target CSVs are treated as **relative transforms**
- the pipeline now resolves relative targets into the effective per-run target before:
  - PEQ fitting
  - predicted-error calculations
  - plotting
  - exported target curves
  - run summaries

This matters most for headphone-to-headphone cloning workflows.

### 2. PEQ fitting now respects the requested filter budget

A filter-budget bug in the PEQ path could allow shelf handling to exceed the requested `max_filters` budget.

0.2.1 fixes this so:
- shelf candidates are budgeted correctly
- `max_filters <= 0` returns no bands cleanly
- the requested band limit is actually respected

### 3. PEQ fitter no longer wastes its search budget on rejected candidates

A second fitter bug could cause obvious underfitting.

The optimizer could:
- repeatedly rediscover a similar candidate
- reject it
- still consume an iteration slot
- stop with only a few bands added even though large residual error remained

0.2.1 fixes this by continuing to search the residual ranking until it finds the next eligible candidate instead of burning the remaining budget on rejected proposals.

This directly improves cases where the generated EQ in 0.2.0 looked far too weak for the measured error.

## User impact

If 0.2.0 gave you a result where:
- clone-target fitting behaved strangely
- the generated EQ looked obviously too weak
- large target mismatch remained even though more filters should have helped

then 0.2.1 is the version to retest.

## Scope

This is a patch release.

It does **not** introduce new workflow modes yet.
In particular:
- conservative up-to-N PEQ remains the current default behavior
- exact-count PEQ and GraphicEQ/fixed-band workflows remain future work

## Upgrade

```bash
python -m pip install --upgrade headmatch
```

Then verify:

```bash
headmatch --version
```

Expected:

```text
headmatch 0.2.1
```
