# HeadMatch 0.8.0 Release Notes

Released: 2026-06-13

## Overview

0.8.0 introduces hearing personalisation — an equipment-free EQ pipeline driven by a
built-in pure-tone hearing test. Users can now generate a personalised EQ preset from
their hearing profile alone, with no measurement microphone required. The release also
contains six bug fixes, including a critical correctness issue in the APO shelf filter
export that has been present since shelf filters were first introduced.

---

## New: Hearing Personalisation Pipeline

### Hearing Test

HeadMatch now ships a self-administered pure-tone audiometric screen based on the
Modified Hughson-Westlake procedure (Carhart & Jerger 1959, public domain) following
the ISO 8253-1 seven-frequency protocol (500 Hz, 1 kHz, 2 kHz, 3 kHz, 4 kHz, 6 kHz,
8 kHz).

The test plays calibrated sine tones through the selected output device. The user
responds by clicking or pressing Enter when a tone is heard. The adaptive "up-5
down-10" staircase declares a threshold when the same level is heard on two of the last
three ascending runs.

The result is saved as a `HearingProfile` in the platform config directory and can be
used immediately or recalled in later sessions.

**CLI:**
```bash
headmatch hearing-test
headmatch hearing-test --fit --out-dir out/hearing_fit
```

**GUI:** Available under the "Hearing Test" navigation item.

### Equipment-Free EQ Pipeline

A `HearingProfile` alone is sufficient to generate a ready-to-load EQ preset. The
pipeline applies the half-gain prescription rule (Lybarger 1944, public domain):

```
loss(f)  = measured_threshold(f) − normal_hearing_reference(f)
gain(f)  = clamp(loss(f) × 0.5, 0, 12 dB)
```

Normal hearing reference values are derived from ISO 7029:2017 median thresholds for
young adults, mapped to the relative dBFS scale used for the test. The seven-point gain
array is interpolated to a full 512-point frequency grid via `CubicSpline` on
log-frequency, then smoothed with 1-octave Gaussian filtering to prevent sharp EQ
transitions.

The pipeline assumes a flat headphone response. For a more accurate result, a headphone
measurement can be taken first and `--with-hearing-compensation` applied during fitting.

**CLI:**
```bash
headmatch hearing-fit --out-dir out/hearing_fit
headmatch hearing-fit --out-dir out/hearing_fit --target-csv targets/harman.csv
```

**GUI:** After a successful hearing test, the completion screen offers "Generate EQ
Preset Now" (equipment-free) or "Run a Measurement" (full measurement + compensation).

## Bug Fixes

### CRITICAL — APO shelf filter exports wrong Q value (`exporters.py`)

Shelf filters written to `equalizer_apo.txt` (LS/HS) carried the raw RBJ slope
parameter S in the Q field instead of the standard shelf Q that Equalizer APO expects.
For a typical S=0.7, headmatch wrote `Q 0.70` when APO's formula requires `Q 0.59`
(derived via `Q = 1/√((A + 1/A)·(1/S − 1) + 2)`). Every exported shelf curve was
measurably wrong in any APO host — the transition-band slope differed by roughly 15%.

The CamillaDSP exporter was already computing the correct value via `band.shelf_q`;
the APO exporter now does the same. Existing presets generated by earlier versions
should be re-exported.

### HIGH — GraphicEQ preamp unnecessarily applied for cut-only EQ (`exporters.py`)

The 1.5 dB interpolation-headroom constant was subtracted unconditionally, so a
pure-cut EQ profile received a -1.5 dB preamp with no benefit. Headroom is now only
applied when the peak gain is above 0 dB, which is the only scenario where APO's
overlapping-band interpolation can cause gain stacking.

### HIGH — Hearing-fit completion screen never showed clipping assessment (`gui/shell.py`)

`_start_hearing_fit` passed the worker return value as `_result` (discarding it) rather
than forwarding it to `_set_completion(result=result)`. The EQ clipping warning was
always absent from the hearing-fit completion screen regardless of boost level.

### HIGH — KeyError crash risk in APO refine status display (`gui/controllers.py`)

`run_apo_refine` used bracket access `report['predicted_left_rms_error_db']` while
every surrounding key was accessed via `.get()`. Any deviation from the expected report
schema would crash the GUI controller with an unhandled `KeyError`. Changed to
`.get('predicted_left_rms_error_db', 0)`.

### HIGH — `_average_measurements` silently dropped diagnostic keys (`pipeline.py`)

The multi-iteration averaging path iterated only `results[0].diagnostics.keys()`,
silently dropping any key present in later results but absent from the first. The key
set is now unioned across all results, making the averaging robust to schema evolution.

### HIGH — Dead conditional in APO refine channel dispatch (`apo_refine.py`)

`MeasurementResult` has no `sample_rate` field, so the guard
`result.sample_rate if hasattr(result, 'sample_rate') else sweep_spec.sample_rate` was
permanently in the fallback. The dead branch made the left-channel dispatch look
inconsistent with the right-channel call directly below it, and would silently diverge
if `MeasurementResult` ever gained `sample_rate`. Removed; both channels now use
`sweep_spec.sample_rate` directly.

## Upgrade Notes

No breaking changes to existing workflows. The hearing test and hearing-fit commands are
additive. Existing measurement and fitting pipelines are unchanged.

If you have previously exported APO parametric presets containing shelf filters (LS/HS),
re-export them from this version to get correct filter curves in Equalizer APO.

```bash
pip install --upgrade headmatch
headmatch --version
# headmatch 0.8.0
```
