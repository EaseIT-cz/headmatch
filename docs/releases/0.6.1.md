# HeadMatch 0.6.1

## Summary

Platform stability release. All test suites pass on Linux and macOS. Standalone binary builds for Linux x64 and macOS ARM64.

## Platform stability

- macOS test suite passes with platform-specific skips for PipeWire-only tests
- XDG config path test skipped on non-Linux (XDG is Linux-specific)
- PortAudio backend test skipped on macOS (requires hardware)
- 520+ deterministic tests across platforms

## Standalone binaries

- **Linux x64**: one-file PyInstaller executable with all dependencies
- **macOS ARM64**: Apple Silicon native with sounddevice for PortAudio
- Intel macOS build removed (all modern Macs are ARM64)

### Binary usage

```bash
# Linux
chmod +x headmatch-0.6.1-linux-x64
./headmatch-0.6.1-linux-x64

# macOS (bypass quarantine)
xattr -d com.apple.quarantine headmatch-0.6.1-macos-arm64
./headmatch-0.6.1-macos-arm64
```

## EQ clipping prediction

- `headmatch/eq_clipping.py` module with `EQClippingAssessment` dataclass
- `assess_eq_clipping()` forecasts whether fitted EQ will cause digital clipping
- Computes required preamp gain (`preamp = -max_positive_boost`)
- Quality warnings for moderate (>6 dB) and severe (>12 dB) headroom loss
- Integrated into fit pipeline — automatically in fit report

## Fixes

- OpenBLAS ELF alignment: pinned numpy<2, scipy<1.14
- Target editor crash in binary: scipy.interpolate added to hiddenimports
- sounddevice bundled in macOS binary for PortAudio

## Tests

- Platform-specific skip markers for macOS
- Backend test mocking for cross-platform CI
- 13 new EQ clipping tests

## Upgrade

```bash
pip install --upgrade headmatch
headmatch --version
# headmatch 0.6.1
```
