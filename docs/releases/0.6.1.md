# HeadMatch 0.6.1

## Summary

Platform stability release. All test suites now pass on Linux and macOS. Standalone binary builds available for Linux x64 and macOS ARM64.

## Platform fixes

- **macOS test suite passes**: platform-specific skips for PipeWire-only tests (device selection, backend tests, doctor checks)
- **XDG config path test**: skipped on non-Linux platforms (XDG is Linux-specific)
- **PortAudio backend test**: skipped on macOS (requires real hardware)
- **Test count**: 520+ deterministic tests across platforms

## Binary builds

- **Linux x64 standalone binary**: one-file PyInstaller executable with all dependencies bundled
- **macOS ARM64 binary**: Apple Silicon native build with sounddevice for PortAudio support
- **Intel macOS build removed**: simplified to ARM64 only (modern Macs are all Apple Silicon)
- **scipy.interpolate fix**: added to hiddenimports (was missing, caused target editor crash in binary)

### Binary usage

```bash
# Linux
chmod +x headmatch-0.6.1-linux-x64
./headmatch-0.6.1-linux-x64

# macOS (may need to bypass quarantine)
xattr -d com.apple.quarantine headmatch-0.6.1-macos-arm64
./headmatch-0.6.1-macos-arm64
```

## Fixes

- OpenBLAS ELF alignment issue resolved by pinning numpy<2, scipy<1.14
- scipy.interpolate added to PyInstaller hiddenimports (fixes target editor crash in binary builds)
- sounddevice bundled in macOS binary for PortAudio backend support

## Test improvements

- `test_default_config_path_prefers_xdg`: skipped on non-Linux with reason
- `test_backend_portaudio`: skipped on macOS (requires hardware)
- `test_doctor_checks`: platform-agnostic via mocking
- `test_format_pipewire_targets`: platform-agnostic via mocking
- Audio device selection mocked in `fake_tk` fixture

- Backend test mocking for cross-platform CI
- EQ clipping prediction (13 tests from 0.6.1rc2)
